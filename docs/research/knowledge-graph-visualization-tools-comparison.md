## 1. 온톨로지 특화 시각화 도구

### 1.1 Protege OntoGraf

**개요:** Stanford 대학에서 개발한 Protege Desktop에 번들된 온톨로지 시각화 플러그인

```
Protege OntoGraf 특성:
┌─────────────────────────────────────────────────┐
│  목적: OWL 온톨로지의 인터랙티브 그래프 탐색     │
│  렌더링: Java Swing 기반                         │
│  레이아웃: Spring, Tree, Radial, Cube 등         │
│  핵심: Asserted vs Inferred 관계 구분            │
│  라이선스: BSD (Protege 번들)                     │
└─────────────────────────────────────────────────┘
```

**핵심 기능:**
- OWL 클래스 계층 인터랙티브 탐색
- 관계 유형별 필터링 (subclass, individual, domain/range, object property, equivalence)
- Asserted(명시적) vs Inferred(추론된) 지식 시각적 구분
- 노드 숨기기/표시 제어
- 다양한 레이아웃 알고리즘 (spring, tree, vertical tree, radial, cube)
- 이미지 형식 내보내기

**강점:**
- Protege와 완전 통합 — 온톨로지 편집 중 즉시 시각화
- OWL 개념(클래스, 속성, 인스턴스, 제약조건)에 최적화
- 별도 설치 불필요 (Protege Desktop 번들)

**한계:**
- 대규모 온톨로지에서 시각적 혼잡 (500+ 노드)
- 인메모리 OWL API 아키텍처로 최대 온톨로지 크기 제한
- 요약/축소 기능 미흡 — 전체 온톨로지를 한 번에 렌더링
- Java Swing 기반으로 웹 배포 불가

**적합 시나리오:** 온톨로지 개발 과정 중 구조 검토, OWL/RDF 교육

### 1.2 WebVOWL

**개요:** VOWL(Visual Notation for OWL Ontologies) 표준을 구현한 웹 기반 온톨로지 시각화 도구

```
WebVOWL VOWL 표기법:
┌─────────────────────────────────────────────────┐
│  ○ 원형 = OWL 클래스                             │
│  □ 사각형 = 데이터타입                           │
│  ─→ 화살표 = 속성 (라벨 포함)                    │
│  ◇ 다이아몬드 = 제약조건 (union, intersection)   │
│  색상 코드: 클래스(파랑), 외부(하늘), 데이터(초록)│
│  레이아웃: Force-directed 자동 배치              │
│  라이선스: MIT                                    │
└─────────────────────────────────────────────────┘
```

**핵심 기능:**
- VOWL 표준 그래픽 표기법 구현 (클래스=원, 속성=화살표, 데이터타입=사각형)
- Force-directed 레이아웃으로 자동 배치
- 속성 유형, 노드 차수 등 다양한 필터링
- 엔티티 상세 정보 표시 (메타데이터, 설명, 메트릭)
- 텍스트 기반 검색
- 설치 불필요 — 브라우저에서 바로 실행

**강점:**
- 직관적인 표준 시각 언어 (VOWL)로 온톨로지 즉시 이해
- 프로그래밍 불필요 — OWL 파일 업로드만으로 시각화 생성
- 웹 기반으로 공유/문서화에 적합
- OWL 구문 요소(클래스, 속성, 인스턴스, 제약조건, union, intersection)를 시각적으로 구분

**한계:**
- 공개 인스턴스 파일 크기 제한 (~1MB)
- SVG 기반으로 대규모 온톨로지에서 성능 저하
- OWL2VOWL 변환기(Java) 필요
- 편집 기능 없음 — 읽기 전용

**최근 개선:** "Making WebVOWL Great Again" (2025) 연구에서 실행 시간 복잡도를 수 배 개선

**적합 시나리오:** 온톨로지 문서화, 웹 기반 공유, OWL 교육, 빠른 구조 파악

### 1.3 Tom Sawyer Perspectives

**개요:** 엔터프라이즈급 그래프 시각화·분석 플랫폼 (로우코드)

```
Tom Sawyer Perspectives 특성:
┌─────────────────────────────────────────────────┐
│  목적: 미션 크리티컬 그래프 시각화 애플리케이션   │
│  아키텍처: 로우코드 개발 플랫폼                   │
│  온톨로지: 네이티브 RDF/SPARQL 지원              │
│  데이터 연합: RDB, 그래프DB, JSON, REST, RDF     │
│  가격: Enterprise 라이선스 (문의)                 │
└─────────────────────────────────────────────────┘
```

**핵심 기능:**
- 5가지 그래프 레이아웃 스타일 (중첩 드로잉, 엣지 라우팅 최적화)
- 내장 그래프 분석 알고리즘 (최단경로, 중심성, 연결성, 클러스터링)
- 자동 스키마 추출
- 코드 없는 패턴 기반 쿼리
- 다중 소스 데이터 연합 (RDB, 그래프 스토어, JSON, RDF, REST API)
- 스키마 에디터로 지식 네트워크 모델링
- Jupyter Notebook 통합
- Web Components 지원

**온톨로지 특화 기능:**
- **네이티브 RDF 데이터소스 통합** (v13.4)
- SPARQL 쿼리 빌더 내장
- OWL 기반 시맨틱 지식 그래프 지원
- 레거시 RDB 위에 시맨틱 레이어 구축 가능

**적합 시나리오:** 사이버보안 위협 분석, 네트워크 인프라 모니터링, 정보 분석, 시스템 엔지니어링

## 2. 범용 그래프 시각화 도구

### 2.1 Neo4j Bloom (Explore)

**개요:** Neo4j 그래프 데이터베이스를 위한 코드리스 시각화 도구

```
Neo4j Bloom 특성:
┌─────────────────────────────────────────────────┐
│  렌더링: GPU 가속 물리/렌더링 엔진               │
│  검색: 유사 자연어 구문 기반 쿼리                │
│  Perspectives: 커스터마이징 가능한 뷰 시스템      │
│  AI: AI co-pilot 통합 (2025~)                    │
│  온톨로지: Neosemantics(n10s) 플러그인           │
│  가격: Aura 무료 / Enterprise 유료               │
└─────────────────────────────────────────────────┘
```

**핵심 기능:**
- "Perspectives" 시스템으로 그래프 데이터의 맞춤형 뷰 정의
- 유사 자연어 검색 (구문 기반 쿼리)
- GPU 기반 물리/렌더링 엔진
- 규칙 기반·조건부 노드/관계 스타일링
- AI co-pilot (2025~) — 자연어로 multi-hop 분석

**온톨로지 연동:**
- Neosemantics(n10s) 플러그인으로 OWL/RDFS/SKOS/RDF 데이터 임포트
- labeled property graph 모델로 변환하여 탐색
- 시맨틱 레이어 기반 추론 지원

**적합 시나리오:** Neo4j 기반 지식 그래프 탐색, 엔터프라이즈 데이터 분석

### 2.2 Gephi

**개요:** 네트워크 과학과 소셜 네트워크 분석에 널리 사용되는 오픈소스 그래프 시각화·분석 플랫폼

```
Gephi 특성:
┌─────────────────────────────────────────────────┐
│  렌더링: OpenGL (0.9.x) / Vulkan (0.10 beta)    │
│  레이아웃: ForceAtlas2, Fruchterman-Reingold,    │
│           Yifan Hu, 기타 다수                    │
│  분석: 모듈성, 중심성, PageRank, 클러스터링      │
│  플러그인: 100+ 커뮤니티 플러그인                │
│  확장성: 50K-200K 노드 (Vulkan: 10M+ edges)     │
│  라이선스: CDDL + GPL                            │
└─────────────────────────────────────────────────┘
```

**핵심 기능:**
- 실시간 시각화 (OpenGL/Vulkan 렌더링 엔진)
- 다양한 레이아웃 알고리즘 (ForceAtlas2, Fruchterman-Reingold, Yifan Hu 등)
- 내장 통계 분석 (모듈성, 매개중심성, PageRank, 클러스터링 계수)
- 동적 그래프 지원 (시간적 네트워크 분석)
- 필터링 및 파티셔닝
- PNG, SVG, PDF, GEXF, GraphML 내보내기

**강점:**
- 대규모 네트워크 처리 능력 (Vulkan 엔진으로 10M+ edges)
- 학술 연구에서 가장 널리 사용되는 도구 중 하나
- 풍부한 통계 분석 내장

**한계:**
- 네이티브 온톨로지 지원 없음 (GraphML/GEXF로 변환 필요)
- 직접 데이터베이스 커넥터 없음
- 학습 곡선이 레이아웃/통계 마스터에 필요

**적합 시나리오:** 학술 연구, 소셜 네트워크 분석, 서지학적 분석, 중대형 네트워크 탐색

### 2.3 Cytoscape

**개요:** 생물학적/분자 네트워크 분석용으로 개발된 오픈소스 플랫폼 (범용 네트워크 시각화 가능)

```
Cytoscape 특성:
┌─────────────────────────────────────────────────┐
│  목적: 생물학적 네트워크 + 속성 데이터 통합       │
│  앱 생태계: 수백 개 도메인 특화 앱               │
│  웹 버전: Cytoscape.js (JavaScript 라이브러리)   │
│  통합: NDEx, STRING, BioGRID, CyREST API        │
│  온톨로지: GO enrichment (BiNGO, ClueGO 앱)     │
│  라이선스: LGPL                                  │
└─────────────────────────────────────────────────┘
```

**핵심 기능:**
- VizMapper — 노드/엣지 속성의 규칙 기반 시각 매핑
- 내장 네트워크 분석 (중심성, 클러스터링, 최단 경로)
- Cytoscape.js — 웹 임베딩용 JavaScript 라이브러리
- NDEx 통합으로 네트워크 공유/영속화
- CX2, CSV, SIF, XGMML, GraphML 등 다양한 포맷 지원
- 레벨-of-detail(LOD) 자동 렌더링

**온톨로지 연동:**
- Gene Ontology(GO) enrichment 분석 (BiNGO, ClueGO 앱)
- OBO/OWL 온톨로지 임포트 (전용 앱)
- 단, 온톨로지 시각화는 부차적 기능

**적합 시나리오:** 바이오인포매틱스, 시스템 생물학, 분자 상호작용 네트워크

### 2.4 D3.js 및 D3 기반 도구

**개요:** 브라우저에서 동적·인터랙티브 데이터 시각화를 생성하는 JavaScript 라이브러리

```
D3.js 기반 그래프 도구 생태계:
┌─────────────────────────────────────────────────┐
│  d3-force: Velocity Verlet 물리 시뮬레이션       │
│  ├── Barnes-Hut 근사 → O(n log n) 성능          │
│  ├── 커스텀 힘 (centering, collision, links,     │
│  │   many-body, positioning)                     │
│  └── SVG / Canvas / WebGL 렌더링 선택            │
│                                                  │
│  Cola.js: 제약 기반 레이아웃                     │
│  ├── 오버랩 방지                                │
│  ├── 방향 그래프 Flow 레이아웃                   │
│  └── D3 force layout 대체 가능                   │
│                                                  │
│  라이선스: ISC (D3) / MIT (Cola.js)              │
└─────────────────────────────────────────────────┘
```

**확장성 비교:**

| 렌더링 방식 | 인터랙티브 노드 한계 | 적합 사례 |
|------------|---------------------|----------|
| D3 + SVG | ~300-1,000 | 소규모 인터랙티브 그래프 |
| D3 + Canvas | ~5,000-10,000 | 중규모 그래프 |
| D3 + WebGL | ~10,000+ | 대규모 그래프 시각화 |
| Cola.js | ~100 (최적) | 제약 기반 소규모 레이아웃 |

**강점:** 완전한 시각적 제어, 브라우저 네이티브, 어떤 웹 프레임워크와도 통합 가능
**한계:** 높은 학습 곡선, 저수준 라이브러리 (즉시 사용 불가), 온톨로지 기능 없음

**적합 시나리오:** 커스텀 웹 그래프 시각화, 대시보드 임베딩, 연구 프로토타이핑

### 2.5 Graphviz

**개요:** AT&T Labs에서 개발한 오픈소스 그래프 시각화 툴킷 (DOT 언어 기반)

```
Graphviz 레이아웃 엔진:
┌─────────────────────────────────────────────────┐
│  dot   : 계층적/방향 그래프 (기본)               │
│  neato : spring model                            │
│  fdp   : force-directed                          │
│  sfdp  : 대규모 force-directed (수만 노드)       │
│  twopi : 방사형                                  │
│  circo : 원형                                    │
│  osage : 배열 기반                               │
│  patchwork : 트리맵                              │
│                                                  │
│  출력: SVG, PNG, PDF, PostScript 등              │
│  라이선스: Eclipse Public License                │
└─────────────────────────────────────────────────┘
```

**핵심 기능:**
- DOT 언어 — 간결하고 사람이 읽을 수 있는 그래프 기술 형식
- 8개 레이아웃 엔진 (계층적, 방사형, 원형, 대규모 force-directed 등)
- 풍부한 출력 형식 (SVG, PNG, PDF, PostScript)
- 서브그래프/클러스터 지원
- 언어 바인딩 (Python, Java, C, Ruby 등)
- 다른 도구의 렌더링 백엔드로 널리 사용

**온톨로지 연동:**
- 네이티브 지원 없으나 래퍼 도구 존재
- `ontology-visualization` (Python): RDF→DOT 변환
- `OntoGraph`: OWL→Graphviz 출력 생성
- `Ontospy`: RDF/OWL 검사·시각화 (Graphviz 렌더러)

**적합 시나리오:** 자동화된 다이어그램 생성, 소프트웨어 문서화, CI/CD 파이프라인 시각화

---

### 2.6 yEd Graph Editor

**개요:** yWorks에서 개발한 무료 크로스플랫폼 그래프 에디터

```
yEd 특성:
┌─────────────────────────────────────────────────┐
│  레이아웃: 계층적, 직교, 유기적, 트리, 원형, 방사형│
│  편집: 드래그 앤 드롭 수동 편집                   │
│  그룹핑: 노드 그룹핑 및 중첩                     │
│  포맷: GraphML(네이티브), GML, XGMML, TGF, Excel│
│  출력: PNG, JPG, SVG, PDF, EMF, GraphML          │
│  웹 버전: yEd Live (브라우저 기반)               │
│  가격: 무료 (yFiles SDK는 상용)                  │
└─────────────────────────────────────────────────┘
```

**강점:** 직관적 GUI, 자동 레이아웃으로 깔끔한 다이어그램 생성, 프로그래밍 불필요
**한계:** 네이티브 온톨로지 지원 없음, 10,000+ 노드에서 레이아웃 계산 30초+

**적합 시나리오:** 범용 다이어그래밍, 플로우차트, UML/ER/BPMN, 출판용 그래프

### 2.7 KeyLines (Cambridge Intelligence)

**개요:** 엔터프라이즈 그래프 시각화 애플리케이션 구축을 위한 상용 JavaScript SDK

```
KeyLines 특성:
┌─────────────────────────────────────────────────┐
│  렌더링: GPU 기반 WebGL → 60fps @ 10K items      │
│  레이아웃: 8가지 자동 레이아웃                    │
│  분석: 최단경로, 중심성, 클러스터링 내장          │
│  특화: 시간 기반 분석, 지리공간 모드              │
│  Combos: 노드 클러스터 그룹핑/접기               │
│  동반 제품: ReGraph (React 컴포넌트)              │
│  가격: ~$4,500+/년 (개발자 라이선스)             │
└─────────────────────────────────────────────────┘
```

**핵심 기능:**
- GPU 기반 WebGL 렌더링 (10K 아이템에서 60fps, 벤치마크 100K)
- 시간 기반 네트워크 진화 분석
- 지리공간 모드 (지도 기반 그래프)
- "Combos"로 노드 클러스터 그룹핑/접기
- Neo4j, Amazon Neptune, JanusGraph, TigerGraph 커넥터

**적합 시나리오:** 사기 탐지, 사이버보안, 법집행, 공급망 분석 (엔터프라이즈 임베딩)

## 5. 온톨로지 특화 시각화 도구 심층 비교

### 5.1 OWL 지원 수준

| 도구 | OWL Class | Object Property | Data Property | Individual | Restriction | Annotation |
|------|-----------|----------------|---------------|-----------|-------------|------------|
| Protege OntoGraf | 완전 | 완전 | 완전 | 완전 | 완전 | 부분 |
| WebVOWL | 완전 | 완전 | 완전 | 부분 | 완전 | 부분 |
| Tom Sawyer | 완전 | 완전 | 완전 | 완전 | 부분 | 부분 |
| Neo4j + n10s | 매핑 | 매핑 | 매핑 | 매핑 | 제한 | 매핑 |
| Cytoscape + 앱 | 부분 | 부분 | 제한 | 제한 | 없음 | 없음 |

### 5.2 추론(Reasoning) 시각화

| 도구 | Asserted 표시 | Inferred 표시 | 추론기 통합 | 불일치 탐지 |
|------|-------------|-------------|-----------|------------|
| Protege OntoGraf | 예 | 예 (Pellet/HermiT) | 직접 통합 | 예 |
| WebVOWL | 예 | 아니오 | 없음 | 아니오 |
| Tom Sawyer | 예 | 제한적 | 외부 연동 | 제한적 |
| Neo4j + n10s | 해당 없음 | 해당 없음 | 없음 | 없음 |

### 5.3 VOWL vs OntoGraf 시각 표기 비교

```
VOWL (WebVOWL):                    OntoGraf (Protege):
┌─────────────────────┐            ┌─────────────────────┐
│  ○ Class (파랑)      │            │  □ Class (기본 노드)  │
│  ◇ Restriction       │            │  ─→ 관계 (엣지)     │
│  □ Datatype (초록)   │            │  색상 = 관계 유형    │
│  ─→ Property (라벨)  │            │  점선 = inferred     │
│                      │            │  실선 = asserted     │
│  표준화된 표기법     │            │  자유형 노드-엣지    │
│  비전문가 친화적     │            │  전문가 친화적       │
└─────────────────────┘            └─────────────────────┘
```
